# 基金页面API集成计划

## 1. 需求分析

### 当前状态
- 基金页面目前完全依靠人工维护数据
- 收益率计算使用本地XIRR算法
- 缺乏实时或最新的基金数据
- 无法直观展示基金的最新收益情况

### 目标
- 集成免费API自动查询基金数据
- 直观展示每个基金的收益情况
- 提高数据的准确性和实时性
- 减少人工维护的工作量
- 默认打开页面保持现状且不查询
- 提供联网查询开关，用户可自行控制是否启用API查询
- 内置多个API，自动更换不可用API
- 当数据解析错误或网络不可用时，不使用默认值或缓存数据

## 2. 技术方案

### 2.1 免费API选择

| API名称 | 数据源 | 优势 | 劣势 |
|---------|---------|---------|---------|
| 天天基金网API | 东方财富 | 数据全面，覆盖基金种类多 | 接口可能不稳定，需要处理反爬 |
| 新浪财经API | 新浪财经 | 接口相对稳定 | 数据更新可能不及时 |
| 聚宽API | 聚宽 | 专业金融数据 | 免费额度有限 |
| 百度股市通API | 百度 | 接口简单易用 | 数据覆盖可能不完整 |

**内置多个API**
- 主API：天天基金网API
- 备用API：新浪财经API
- 实现自动故障转移机制，当主API不可用时自动切换到备用API

### 2.2 实现方案

#### 2.2.1 前端修改
1. **添加联网查询开关**
   - 在基金数据维护按钮左侧增加一个联网查询的开关按钮
   - 开关默认关闭，打开页面时保持现状且不查询
   - 开启后才会调用API查询数据

2. **添加API查询功能**
   - 当联网查询开关开启时，通过基金代码自动查询基金信息
   - 为现有基金添加手动刷新数据功能

3. **扩展表格列**
   - 历史数据基金列表不需要修改
   - 当前基金表格：开启联网查询后，在本金金额与交易详情之间插入相应的列
   - 添加"最新净值"、"日涨跌幅"、"近1年收益"、"7日年化"等列
   - 根据基金类型动态显示相关指标（如货币基金显示7日年化，股票基金显示日涨跌幅等）

4. **添加数据更新机制**
   - 只有当联网查询开关开启时才会更新数据
   - 添加手动刷新按钮
   - 实现定时自动更新（如每30分钟），但只在开关开启时执行

#### 2.2.2 后端修改
1. **API调用服务**
   - 创建基金数据查询服务
   - 实现API请求和数据解析
   - 处理API调用限制和错误重试

2. **数据缓存**
   - 缓存API响应数据，减少重复请求
   - 设置合理的缓存过期时间（如1小时）

3. **数据库扩展**
   - 为基金表添加字段：latestNetValue, dailyChange, yearlyReturn等

### 2.3 数据流设计

1. **初始化流程**
   - 页面加载时，获取所有基金记录
   - 对每个基金调用API获取最新数据
   - 更新本地数据库和页面显示

2. **手动刷新流程**
   - 用户点击刷新按钮
   - 对选中基金或所有基金调用API
   - 更新本地数据和页面显示

3. **添加基金流程**
   - 用户输入基金代码
   - 系统自动查询基金信息
   - 填充基金名称、类型、风险等级等信息
   - 用户确认后保存

## 3. 具体实现步骤

### 3.1 前端修改

1. **修改基金表格结构**
   - 在`tabs/fund.html`中为当前基金表格添加新列
   - 在`tabs/fund.html`中为历史基金表格添加新列

2. **修改fund.js**
   - 添加API查询函数
   - 修改表格渲染函数，支持新列显示
   - 添加数据更新机制
   - 添加刷新按钮事件处理

3. **添加样式**
   - 在`style.css`中为新列添加样式
   - 为涨跌幅添加颜色标识（红色上涨，绿色下跌）

### 3.2 后端修改

1. **创建API服务**
   - 在`routes/`目录下创建`fundApi.js`
   - 实现基金数据查询接口

2. **修改数据库结构**
   - 更新基金表结构，添加新字段
   - 修改相关API接口，支持新字段

3. **实现缓存机制**
   - 添加内存缓存或文件缓存
   - 实现缓存过期策略

### 3.3 错误处理

1. **API调用错误**
   - 网络错误：显示错误提示，不使用缓存数据
   - 接口限制：实现请求频率控制，自动切换到备用API
   - 数据解析错误：不使用默认值或缓存数据，显示错误提示

2. **数据展示错误**
   - 当API数据不可用时，不显示相关列或显示空白
   - 当网络不可用时，不使用缓存数据，显示错误提示
   - 实现自动故障转移机制，当主API不可用时自动切换到备用API

## 5. 风险评估

### 5.1 潜在风险

1. **API稳定性**
   - 天天基金网API可能随时变更
   - 可能面临反爬限制

2. **数据准确性**
   - API返回的数据可能与实际有差异
   - 网络延迟可能导致数据不同步

3. **性能影响**
   - 大量基金数据查询可能影响页面加载速度
   - 频繁API调用可能触发限制

### 5.2 风险缓解措施

1. **API稳定性**
   - 实现多API源备份
   - 定期检查API可用性
   - 实现错误重试机制

2. **数据准确性**
   - 交叉验证多个数据源
   - 保存历史数据用于比对
   - 提供手动修正功能

3. **性能影响**
   - 实现数据缓存
   - 分批查询基金数据
   - 优化API调用策略
